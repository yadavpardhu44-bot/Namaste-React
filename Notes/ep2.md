# Episode 2 – Theory Notes

## 1. What is NPM?

It is the default package manager for Node.js used to **install, manage, and share JavaScript packages (libraries)**.

Key points:

* Helps install third-party packages.
* Manages project dependencies.
* Maintains project configuration using `package.json`.

Example command:

```bash
npm install react
```

---

# 2. What is Parcel / Webpack? Why do we need it?

Parcel and Webpack are **module bundlers**.

A module bundler takes multiple files (HTML, CSS, JS, images) and **bundles them into optimized files** for the browser.

Why we need bundlers:

* Combine multiple files into one bundle
* Optimize code for production
* Improve performance
* Enable modern JavaScript features
* Manage dependencies

Example bundlers:

* Parcel
* Webpack
* Vite

Parcel is beginner-friendly because it **requires zero configuration**.

---

# 3. What is `.parcel-cache`?

`.parcel-cache` is a **cache folder created by Parcel**.

Purpose:

* Stores build cache
* Speeds up rebuilds
* Avoids recompiling unchanged files

This folder **should not be pushed to GitHub** and must be added to `.gitignore`.

---

# 4. What is `npx`?

`npx` is a tool that comes with npm used to **execute packages without installing them globally**.

Example:

```bash
npx parcel index.html
```

This runs Parcel directly from the project dependencies.

Benefits:

* No global installation required
* Runs correct project version
* Prevents version conflicts

---

# 5. Difference between `dependencies` and `devDependencies`

| dependencies                       | devDependencies                  |
| ---------------------------------- | -------------------------------- |
| Required for production            | Required only during development |
| Needed for running the application | Needed for development tools     |
| Installed with `npm install`       | Installed with `npm install -D`  |
| Example: React                     | Example: Parcel, Webpack         |

Example:

```bash
npm install react
```

```bash
npm install -D parcel
```

---

# 6. What is Tree Shaking?

Tree Shaking is a **process of removing unused code from the final bundle**.

It helps reduce bundle size and improves performance.

Example:

If you import only one function from a library, unused functions are removed during the build.

Benefits:

* Smaller bundle size
* Faster loading
* Better performance

---

# 7. What is Hot Module Replacement (HMR)?

Hot Module Replacement allows **updating modules in the browser without refreshing the entire page**.

When you change code, the browser automatically updates only the changed module.

Benefits:

* Faster development
* Keeps application state
* Instant feedback while coding

Parcel provides HMR automatically.

---

# 8. Favourite 5 Superpowers of Parcel

1. Zero configuration setup
2. Fast bundling using caching
3. Hot Module Replacement (HMR)
4. Tree shaking
5. Production optimization

### Description of 3 Superpowers

**1. Zero Configuration**

Parcel works without complex configuration files. Developers can start a project immediately.

**2. Hot Module Replacement**

Automatically updates modules in the browser when code changes without refreshing the page.

**3. Fast Build with Caching**

Parcel uses caching to store previous builds, making rebuilds extremely fast.

# 9. What is `.gitignore`?

`.gitignore` is a file used to **tell Git which files or folders should not be tracked**.

Example:

```
node_modules
dist
.parcel-cache
.env
```

These files are ignored because they are either:

* large
* auto-generated
* system-specific

---

# 10. Difference between `package.json` and `package-lock.json`

| package.json                     | package-lock.json                  |
| -------------------------------- | ---------------------------------- |
| Contains project metadata        | Contains exact dependency versions |
| Written and edited by developers | Automatically generated by npm     |
| Defines dependency ranges        | Locks dependency versions          |

`package-lock.json` ensures **consistent installations across machines**.

---

# 11. Why should we not modify `package-lock.json`?

`package-lock.json` is automatically generated by npm to maintain **exact dependency versions**.

Manually modifying it may cause:

* dependency conflicts
* installation errors
* inconsistent builds

Therefore it should be **managed only by npm**.

---

# 12. What is `node_modules`?

`node_modules` is a folder where **all installed project dependencies are stored**.

Example packages inside it:

* React
* Parcel
* Lodash

It can become very large.

### Should we push it to GitHub?

No.

Reasons:

* It is very large
* It can be regenerated using:

```bash
npm install
```

Therefore it should be added to `.gitignore`.

---

# 13. What is the `dist` folder?

`dist` stands for **distribution folder**.

It contains the **optimized production build of the application**.

Example contents:

* bundled JavaScript
* minified CSS
* optimized assets

This folder is generated when running:

```bash
npm run build
```

---

# 14. What is `browserslist`?

`browserslist` defines **which browsers your project should support**.

Tools like Parcel use this to determine:

* which JavaScript features to transpile
* which polyfills are needed

Example:

```
"browserslist": [
  "last 2 versions",
  ">1%"
]
```

# 16. `^` (Caret) vs `~` (Tilde)

These symbols define **version ranges in dependencies**.

Example:

```
^1.2.3
```

Allows updates for **minor and patch versions**.

Example:

```
~1.2.3
```

Allows updates only for **patch versions**.

Summary:

| Symbol | Allows Updates |
| ------ | -------------- |
| ^      | minor + patch  |
| ~      | patch only     |

# 17. Script Types in HTML

The `type` attribute in `<script>` defines how the script should be interpreted.

Common types:

### JavaScript

```
<script src="app.js"></script>
```

### Module Script

```
<script type="module" src="app.js"></script>
```

Module scripts allow:

* ES6 imports and exports
* strict mode
* modular JavaScript

Example:

```javascript
import React from "react";
```
